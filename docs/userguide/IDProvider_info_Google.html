HybridAuth, Open Source Social-Sign-On Solution


A PHP Library for authentication through Facebook, Twitter, Google, Yahoo, MySpace, LinkedIn, FourSquare, OpenID and many other Identity providers

HybridAuth User guide » Supported identity providers » Google

Synopsis


<?php

    # start a new PHP session

    session_start();



    # include HybridAuth main file

    include "path/to/hybridauth/hybridauth.php";




    # create a new Hybrid_Auth instance 

    $hybridauth = new Hybrid_Auth();

        

    # setup Google adapter 


    $provider_adapter = $hybridauth->setup( "Google" );



    # start login

    $provider_adapter->login();

?>

Registering application

Go to Manage your domains, and fill required input as showen in this screenshot. then copy the created application credentials (key and secret) and put them on HybridAuth config file. Make sure to put your correct website adress in the "Target URL path prefix" field.

Provider Adapter

 ID Google
 Protocol OpenID + OAuth
 IDp URL http://www.google.com/
 Keys registeration http://code.google.com/apis/accounts/docs/RegistrationForWebAppsAuto.html
 Dev documentation http://code.google.com/apis/accounts/docs/OpenID.html
 Description Google now supports a hybrid protocol, combining Federated Login with OAuth (OpenID+OAuth). Many third-party sites enable OpenID to let their users log in with their Google account information (freeing the third-party developer from having to manage login security).
 Author Zachy <hybridauth=at=gmail-dot-com>
 Based on api.eric work code.google.com/p/gdata-samples/../hybrid 
 Version 1.1
 Since HybridAuth 1.0.1
 Wrapper ./Hybrid/Providers/Google.php
 Required Libs ./Hybrid/thirdparty/Auth/
 URL Start login* http://mywebsite.com/path_to_hybridauth/?hauth.start=Google
 URL Login done* http://mywebsite.com/path_to_hybridauth/?hauth.done=Google

* For more informations about adapter URL Start Login and URL Login done, please refer to HybridAuth endpoint URL section.
