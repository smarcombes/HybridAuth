HybridAuth, Open Source Social-Sign-On Solution



A PHP Library for authentication through Facebook, Twitter, Google, Yahoo, MySpace, LinkedIn, FourSquare, OpenID and many other Identity providers

HybridAuth User guide » Setup and Configuration of Supported Identity Providers

1. Supported providers

HybridAuth currently supports the following external providers:
  • OpenID
  • Facebook
  • LinkedIn
  • Google (Gmail, Youtube, Orkut, ...)
  • Twitter
  • Windows Live
  • MySpace
  • Foursquare
  • Friendster
  • Vimeo
  • Yahoo
  • PayPal
  • Tumblr
  • Gowalla
  • AOL (AOL API Seems to be broken)

2. Setup Providers Applications Keys

Before the HybridAuth can work, you need to request an external applications for each social networks and identity providers you wish to use in your web site from theirs [IDps] web sites. These external applications ensures that users are logging into the proper Web site and allows social networks or the identity providers to send the user back to the correct Web site after successfully authenticating their Accounts.

Your applicatoins API keys must be set on hybridauth/hybridauth.php configuration file: 


<?php

    /**

    * List of enabled IDps and IDps credentials

    *

    * Note: You have to fill your applications credentials ( consumer, secret, appid, etc . )


    *       To know how to register your application and get keys please refer to the userguide

    *       http://hybridauth.sf.net/userguide/Supported_identity_providers_and_setup_keys.html


    */

    $GLOBAL_HYBRID_AUTH_IDPROVIDERS = ARRAY

    (

            "Twitter"

                    => ARRAY

                    (

                            "enabled"       => TRUE,

                            "keys"          => ARRAY

                            (


                                    "CONSUMER_KEY"    => "3Xq2hqLhP6lTU2Qh0RUeA",

                                    "CONSUMER_SECRET" => "********************************",

                            ),

                            "wrapper"       => "Providers/Twitter.php"

                    ),

     

            "Facebook"

                    => ARRAY


                    (

                            "enabled"       => TRUE,

                            "keys"          => ARRAY

                            (

                                    "APPLICATION_ID"  => "185915804256",

                                    "CONSUMER_SECRET" => "*****************************",


                            ),

                            "wrapper"       => "Providers/Facebook.php"

                    ),

     

            // and so on ...

    );

?>
