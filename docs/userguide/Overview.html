1. Introduction

HybridAuth is an open source web based single-sign-on solution. HybridAuth allow and manage the authentication process when a user want to log into your web site or blog with their OpenID, Facebook, Google, Twitter, MySpace, AOL and some other social networks or IDentity Providers services (also called IDp).

The main task of HybridAuth is to performs two types of actions: user Authentication and access Autorisation:

i) During the Authentication step, the user is redirected to his identity provider authentification page to enter their login and password and be asked for approval. When the user is returned back to your web site, the identity provider sends an encrypted token that indicates a successful Authentication.

ii) Once a user has authenticated, HybridAuth move to the next step and requires that the identity provider give your web site a temporary Autorisation to read the user account information (profile, contacts list, etc.).


2. keep it short, What is HybridAuth?

HybridAuth is A PHP Library for authentication through Facebook, Twitter, Google, Yahoo, MySpace, LinkedIn, FourSquare, OpenID and many other Identity providers. That is it!


3. why bother?

  • HybridAuth makes it easy to authenticate users with a wide variety of well known social websites,
  • After authenfication, HybridAuth provide the connected user profile's in a rich, simple and standardized structure across all the social APIs,
  • Been made on top of others open source third party client libraries, HybridAuth allow you to use thoses libraries not only to autenticates users but also for interacting with the social API client the user is connected to. (eg. you can grab the user freinds list, events list, the user photos or albums list, update status, to upload stuff, to edit some other stuff, ... ),
  • The complexity of implementing identity providers authentication and authorization protocols are simplified for the developers,
  • All transactions are completely transparent to the end user,

And also, to get HybridAuth up and running, all what you have to do, is to install HybridAuth library in your site, configure some required Social Network extarnal applications Keys, add a couple of lines and some awesome "sign in with ..." icons to your login page, and it's done.


4. How it work?

  1. From your login page, a user chooses to "sign-in with" an IDp,
  2. HybridAuth will handle the authentication request by instantiating the appropriate IDp adapter,
  3. Once started, the adapter will redirect the user to the chosen IDp site, where it will be asked for approval,
  4. IF the user don't grants access to your site, some IDp will callback (redirect the user once again to your web site) in what they call "cancel mode". JUMP to .7
  5. ELSEIF the user grants access to your site, IDp will callback again your site with some extra GET/POST parameters attached,
  6. HybridAuth, will do some magic to grap the user profile from the IDp API,
  7. End of story.

5. Why?

After reading a stupid blog post about federated login, my boss has asked me to allow our users to login to our website with different providers like facebook, twitter but without using any of the already free to use service out there ...



Back then, I almost passed away while reading and trying to understand the so long and the SO boring developer documentation of each social network api and each authentification protocol specification. So, HybridAuth come as a try to prevent this from happening to another one for the sake of humanity :D
