HybridAuth, Open Source Social-Sign-On Solution


A PHP Library for authentication through Facebook, Twitter, Google, Yahoo, MySpace, LinkedIn, FourSquare, OpenID and many other Identity providers

HybridAuth User guide » Access Social API clients


Been made on top of others open source third party client libraries, HybridAuth allow you to use thoses libraries not only to autenticates users but also for interacting with the social API client the user is connected to, (eg. you can grab the user freinds list, events list, the user photos or albums list, update or get status, to upload stuff, to edit some other stuff, ... ).

HybridAuth allow you to access the user data and the provider client in very simple way. All what you have to do is to call Provider_adapter::user() for the user data and Provider_adapter::api() for the client.

To use this feature, you have to know how the original IDps API Clients work, and this by reading their respectives developers docs ("Based on" field on the Provider Adapter section).

Note: A generic function to update user status and to grab his friends list (across all social APIs) will be made available in further releases.

 
<?php

# We assume the user is already connected to twitter

    $hybridauth = new Hybrid_Auth(); # create the main HybridAuth instance


    $provider_adapter = $hybridauth->wakeup(); # callback the provider adapter

    

    # just like we access the current user profile  


    $user_data = $provider_adapter->user();

    

    # we use api() to access TwitterOAuth object based on Abraham lib :

    # github.com/abraham/twitteroauth 


    $twitter_client = $provider_adapter->api(); 

?>
